Gridea和github搭建博客记录

发布于 2020-07-02 | 分类于 gridea 、 github 、 博客搭建 | 3分钟 | 591字数

来来回回，换了很多网站与工具。最终选择了Gridea和github来搭建自己的博客，一来只是把它作为一个工具，记录下自己的算是确定下来了吧。

在我搭建博客的过程中，来来回回换了很多网站与工具。也走了很多弯路，最终选择了Gridea和github来搭建自己的博客，记录自己的学习过程。之所以没有选择程序员主流的hexo，【虽然我也用过】。一来只是单纯的把它作为一个工具，二来对于非科班出身的我还有其他事情要做。gridea初始化配置简单，后期维护也很方便。非常适合像我这样的散人。

今天，就重新复现一下创建过程，省的以后在忘，留个教程。

1.本地安装

git安装

Gridea 的同步功能依赖本地 Git 客户端，所以我们需要先安装 Git。 Git官网下载平台对应的安装程序。安装没有特别的要求，一路下一步即可。

不过gridea v0.9.2版本以上集成了git，可以只安装gridea软件。

Gridea安装

Gridea官网下载对应平台的安装程序进行安装。

网速较慢，推荐去各大资源站下载。找不到附我的资源库：油麦同学的藏宝阁

2.github仓库创建

1.账户注册

github官网 注册个账户，记好账户与密码。

需要验证你的邮箱

验证完之后，选择第一个，创建一个仓库。

或者返回主界面，选择这个

2.创建仓库

网站就算建好了，但是里面没有东西。输入网站可查看：https://ym777.github.io/

接下来生成令牌：

主界面下，找到Settings>Devloperr settings>Peronal access tokens> Generate new token

生成的token

3.上传到Gridea

安装gridea后打开，按照我的填写内容，填写你的信息。

点保存，点检测远程连接。会出现连接成功字样。点击同步，同步成功，等待片刻。文件就会上传到github托管。

输入自己的域名就能看到更改了。

4.其他功能

写文章可以用markdown，支持他的语法，然后保存点同步就行了。不需要其他知识。

其他功能自行摸索。
